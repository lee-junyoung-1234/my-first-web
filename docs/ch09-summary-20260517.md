로컬에서 수행한 절차
1. 의존성 설치: `npm install`
2. 빌드 확인: `npm run build` (빌드 문제 해결하여 통과)
3. 개발 서버 실행: `npm run dev` → http://localhost:3000
4. 엔드포인트 검증:
   - `GET /login`, `GET /signup`, `GET /posts/new` → 200
   - `POST /api/posts` (JSON body) → 200, 응답 `{"ok":true,"id":4}`

환경변수 및 배포 관련
- 로컬 `.env.local`에 다음 값을 추가(로컬 테스트용):
  - `NEXT_PUBLIC_SUPABASE_URL=https://idwcmkeomvzexkdyectq.supabase.co`
  - `NEXT_PUBLIC_SUPABASE_ANON_KEY=<anon key inserted today>`
  - `.env.local`은 레포에 커밋하지 않았음
- Vercel 환경변수: `NEXT_PUBLIC_SUPABASE_URL`, `NEXT_PUBLIC_SUPABASE_ANON_KEY` CLI로 추가함
- Vercel 배포 URL: https://my-first-web-azure-three.vercel.app (배포 확인됨, HTTP 200)

중요한 주의사항
- `NEXT_PUBLIC_SUPABASE_ANON_KEY`는 공개 키이므로 브라우저에서 보일 수 있음(anon 키는 클라이언트용). `service_role` 키는 절대 클라이언트에 두지 마세요.
- 현재 코드에서 Supabase 클라이언트는 실환경으로 복원되어 있으며, 실제 회원가입/로그인 흐름은 Supabase 대시보드의 Auth 설정(이메일 템플릿, 리디렉션 URL 등)에 따라 동작합니다.
